---
description: "Core design principles: root cause, minimal fixes, small interfaces, zero-value structs"
---
<principles>
- fix_root: address root cause
- minimal_fix: prefer one-line fixes
- no_overabstract: avoid over-abstraction
- single_responsibility: one function, one task
- zero_value: design structs with useful zero values
- small_interfaces: keep interfaces to 1-2 methods
- consistent_receivers: all methods use pointer if any does
</principles>
