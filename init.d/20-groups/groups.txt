# Groups every deploy user is added to unconditionally.
#
# Only baseline / OS-level groups belong here — groups that should
# exist on every host regardless of which capabilities are enabled.
#
# Capability-specific groups are owned by their respective steps:
#   docker  → 50-docker/run.sh  (docker)
#   kvm     → 57-kvm/run.sh     (kvm, libvirt)
#
# `systemd-journal` lets the deploy user read `journalctl` without
# sudo. `adm` is the historical "can read logs" group on Debian/Ubuntu.
#
# Re-running is idempotent (usermod -aG; groupadd -f is a no-op when
# the group already exists).
